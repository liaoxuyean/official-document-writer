# Document Types and Usage (公文文种与使用)

## Overview

According to "党政机关公文处理工作条例", there are 15 types of official documents.

## Document Types

### 1. 决定 (Decision)

**定义：** 适用于对重要事项作出决策和部署、奖惩有关单位和人员、变更或撤销下级机关不适当的决定事项。

**使用场景：**
- 重要决策部署
- 表彰先进单位和个人
- 批准重大事项
- 变更或撤销下级不当决定

**格式特点：**
- 标题一般为"关于……的决定"
- 正文一般包括决定原因、决定事项、执行要求
- 可标注主送机关，也可不标注

**示例结构：**
```markdown
关于XXX的决定

各XX、XX：

[决定缘由]

[决定事项]

[执行要求]

XX（发文机关）
XXXX年X月X日
```

---

### 2. 命令/令 (Order)

**定义：** 适用于公布行政法规和规章、宣布施行重大强制性措施、批准授予和晋升衔级、嘉奖有关单位和人员。

**使用场景：**
- 公布法律、法规
- 发布行政命令
- 任免、嘉奖
- 宣布戒严

**格式特点：**
- 发文机关标志用全称或规范化简称
- 无主送机关
- 签发人职务和姓名

**示例结构：**
```markdown
中华人民共和国XX部令
第X号

[命令内容]

部长 XXX
XXXX年X月X日
```

---

### 3. 公告 (Public Notice)

**定义：** 适用于向国内外宣布重要事项或者法定事项。

**使用场景：**
- 向国内外宣布重要事项
- 公布法定事项
- 宣布重大决定

**格式特点：**
- 发文机关一般不标注
- 无主送机关
- 标题一般为"XX公告"

---

### 4. 通告 (Announcement)

**定义：** 适用于在一定范围内公布应当遵守或者周知的事项。

**使用场景：**
- 公布社会周知事项
- 公布遵守事项
- 告知公众特定事项

**格式特点：**
- 可标注主送机关
- 可不标注主送机关

---

### 5. 通知 (Notice) - 最常用

**定义：** 适用于发布、传达要求下级机关执行和有关单位周知或者执行的事项，批转、转发公文。

**使用场景：**
- 批转下级公文
- 转发上级公文
- 部署工作
- 通知事项
- 任免人员

**分类：**
1. 指示性通知 - 部署工作、传达指示
2. 批转性通知 - 批转下级公文
3. 转发性通知 - 转发上级公文
4. 事项性通知 - 告知具体事项
5. 任免通知 - 任免干部

**格式特点：**
- 标题一般为"关于……的通知"
- 有主送机关
- 正文包括通知缘由、通知事项、执行要求

**示例结构：**
```markdown
关于XXX的通知

各XX：

[通知缘由]

[通知事项]

[执行要求]

XX（发文机关）
XXXX年X月X日
```

---

### 6. 通报 (Circular)

**定义：** 适用于表彰先进、批评错误、传达重要精神和告知重要情况。

**使用场景：**
- 表彰先进单位和个人
- 批评错误行为
- 传达重要精神
- 告知重要情况

**分类：**
1. 表彰性通报
2. 批评性通报
3. 情况通报

---

### 7. 议案 (Proposal)

**定义：** 适用于各级人民政府按照法律程序向同级人民代表大会或者人民代表大会常务委员会提请审议事项。

**使用场景：**
- 提请审议法规
- 提请审议重大事项
- 提请任免

**格式特点：**
- 主送机关为人大或人大常委会
- 签署政府首长姓名

---

### 8. 报告 (Report)

**定义：** 适用于向上级机关汇报工作、反映情况，回复上级机关的询问。

**使用场景：**
- 汇报工作情况
- 反映问题
- 回复上级询问
- 提出意见建议

**分类：**
1. 工作报告 - 汇报工作进展
2. 情况报告 - 反映重要情况
3. 答复报告 - 回复上级询问

**格式特点：**
- 标题一般为"关于……的报告"
- 主送机关为上级机关
- 结尾用"特此报告"或"以上报告，请审阅"
- 不需要上级批复

**示例结构：**
```markdown
关于XXX的报告

XX（上级机关）：

[报告缘由]

[报告主体]
一、……
二、……
三、……

[结语]
特此报告。

XX（发文机关）
XXXX年X月X日
```

---

### 9. 请示 (Request)

**定义：** 适用于向上级机关请求指示、批准。

**使用场景：**
- 请求指示
- 请求批准
- 请求帮助

**分类：**
1. 求示性请示 - 请求指示
2. 求准性请示 - 请求批准
3. 求助性请示 - 请求帮助

**格式特点：**
- 标题一般为"关于……的请示"
- 主送机关只能有一个
- 结尾用"妥否，请批示"或"当否，请批示"
- 需要上级批复
- 可标注签发人

**示例结构：**
```markdown
关于XXX的请示

XX（上级机关）：

[请示缘由]

[请示事项]

[请示结语]
妥否，请批示。

XX（发文机关）
XXXX年X月X日
```

---

### 10. 批复 (Reply)

**定义：** 适用于答复下级机关请示事项。

**使用场景：**
- 答复请示事项
- 批准或否定下级请求

**格式特点：**
- 标题一般为"关于同意/不同意……的批复"
- 主送机关为请示机关
- 正文包括批复依据、批复意见
- 结尾用"此复"

**示例结构：**
```markdown
关于XXX的批复

XX（请示机关）：

你单位《关于XXX的请示》（X〔XXXX〕X号）收悉。
经研究，[批复意见]。

此复。

XX（发文机关）
XXXX年X月X日
```

---

### 11. 意见 (Opinion)

**定义：** 适用于对重要问题提出见解和处理办法。

**使用场景：**
- 提出工作建议
- 指导下级工作
- 提出处理办法

**格式特点：**
- 可上可下可平行
- 标题一般为"关于……的意见"

---

### 12. 函 (Letter)

**定义：** 适用于不相隶属机关之间商洽工作、询问和答复问题、请求批准和答复审批事项。

**使用场景：**
- 商洽工作
- 询问问题
- 答复问题
- 请求批准
- 答复审批

**分类：**
1. 商洽函 - 商洽工作
2. 询问函 - 询问问题
3. 答复函 - 答复问题
4. 请批函 - 请求批准

**格式特点：**
- 使用信函格式
- 发文字号在右上角
- 结尾用"特此函复"或"请予函复"

**示例结构：**
```markdown
XX关于XXX的函

XX：

[函告缘由]

[函告事项]

[函告结语]
特此函告。

XX（发文机关）
XXXX年X月X日
```

---

### 13. 纪要 (Minutes)

**定义：** 适用于记载会议主要情况和议定事项。

**使用场景：**
- 记载会议情况
- 记录议定事项
- 传达会议精神

**格式特点：**
- 标题一般为"XX会议纪要"
- 无主送机关
- 正文包括会议概况、会议内容、议定事项

---

### 14. 决议 (Resolution)

**定义：** 适用于经会议讨论通过的重要决策事项。

**格式特点：**
- 经会议讨论通过
- 标注会议名称和日期

---

### 15. 公报 (Communiqué)

**定义：** 适用于公布重要决定或者重大事项。

**格式特点：**
- 无主送机关
- 标题一般为"XX公报"

---

## Document Selection Guide

| Purpose | Document Type |
|---------|---------------|
| 部署工作 | 通知、决定、意见 |
| 请求批准 | 请示、函 |
| 汇报工作 | 报告 |
| 答复请示 | 批复 |
| 表彰批评 | 通报、决定 |
| 商洽工作 | 函 |
| 会议记录 | 纪要 |
| 公布事项 | 公告、通告 |
